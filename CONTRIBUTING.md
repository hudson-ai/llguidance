# Contributing to llguidance

## Building

```bash
# Build the Rust workspace
cargo build

# Build in release mode
cargo build --release

# Build just the parser crate
cargo build -p llguidance
```

## Testing

```bash
# Run Rust tests (debug)
cargo nextest run

# Run Rust tests (release)
cargo nextest run --release

# Run Python tests (after installing)
pip install -v -e .
pytest
```

## Linting

```bash
# Check formatting
cargo fmt --check

# Run clippy
cargo clippy --workspace --all-targets --all-features -- -D warnings

# Check that the crate builds without default features
cargo check --no-default-features -p llguidance
```

## C/C++ header

The C/C++ header `parser/llguidance.h` is generated by
[cbindgen](https://github.com/mozilla/cbindgen) from the Rust FFI source code.
**If you modify any `#[no_mangle]` or `extern "C"` functions** (or their doc
comments), you must regenerate the header and commit it:

```bash
cargo check -p llguidance --features generate-header
```

CI enforces that the committed header is up to date — if you forget this step,
the "Check cbindgen" lint job will fail.
